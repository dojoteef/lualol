Module lol.api

Basic access to the League of Legends API

This module is one of the core modules that everything is built on top of.

If you want to use any of the premade endpoint modules such as 'lol.summoner' you will need to create an api object first and then use that to create the specific endpoint module.

api (keyfile, regionid, cachedir, options)
Create a new api object which is bound to a region and a specific cache

Parameters:

  • keyfile
    • location of the file with the League of Legends API Key in it
  • regionid
    • the id of the region to speak to (see https://developer.riotgames.com/docs/regional-endpoints)
  • cachedir
    • a directory to cache responses from the API (NOTE: a subdirectory will be made for the region inside the cache directory)
  • options
    • a table with optional parameters, currently the only optional parameter is setting verbose to true

Returns:

    a new api object
_api.isvalid (obj)
Check if the passed in table is a valid api object

Parameters:

  • obj
    • object to validate

Returns:

    true if valid, false otherwise
_api:get (pathTemplate, pathParams, optional)
Make a get request to the League of Legends API

Parameters:

  • pathTemplate
    • a string template using ${variable} for replaceable components
  • pathParams
    • the values of the variables used in the template, note that ${region} and ${platformId} are automatically filled in so no need to specify them
  • optional
    • optional values such as the callback function and query table

Returns:

    true if valid, false otherwise
