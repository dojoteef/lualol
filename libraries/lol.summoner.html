Module lol.summoner

Allows for making queries against the League of Legends Summoner API

By first creating an api object and creating a new summoner object from it you can then make queries against the League of Legends Summoner API.

Functions

summoner (apiObj) Create a new summoner object
_summoner.standardizeSummonerName (summonerName) Given a Summoner name, put it into the League of Legends API standardized format
_summoner:getByName (name, callback) Get a Summoner from the League of Legends API given a Summoner name
_summoner:getByNames (names, callback) Get multiple Summoners from the League of Legends API given their Summoner names
_summoner:getByIds (ids, filter, callback) Get multiple Summoners from the League of Legends API given their Summoner ids

summoner (apiObj)
Create a new summoner object

Parameters:

  • apiObj
    • the api object that communicates with the League of Legends server

Returns:

    a new summoner object
_summoner.standardizeSummonerName (summonerName)
Given a Summoner name, put it into the League of Legends API standardized format

    • the Summoner name to standardize

Returns:

    a Summoner name in the League of Legends API standardized format
_summoner:getByName (name, callback)
Get a Summoner from the League of Legends API given a Summoner name

Parameters:

  • name
    • the Summoner name of the Summoner to retreive
  • callback
    • a callback which receives the response from the API
_summoner:getByNames (names, callback)
Get multiple Summoners from the League of Legends API given their Summoner names

Parameters:

  • names
    • an array-like table with the list of Summoner names to retreive
  • callback
    • a callback which receives the response from the API
    • NOTE: Since you may only retreive 40 summoners at a time the callback may
    • be called multiple times from a single call to summoner:getByNames
_summoner:getByIds (ids, filter, callback)
Get multiple Summoners from the League of Legends API given their Summoner ids

Parameters:

  • ids
    • an array-like table with the list of Summoner ids to retreive
  • filter
    • optional parameter denoting what type of information to retreive
    • valid values: 'name', 'masteries', 'runes'
  • callback
    • a callback which receives the response from the API
    • NOTE: Since you may only retreive 40 summoners at a time the callback may
    • be called multiple times from a single call to summoner:getByIds
