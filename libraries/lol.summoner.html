Module lol.summoner

Allows for making queries against the League of Legends Summoner API

By first creating an api object and creating a new summoner object from it you can then make queries against the League of Legends Summoner API.

Class summoner

This class encapsulates manipulating the League of Legends Summoner API
summoner:summoner (api)
Create a new summoner object

Parameters:

  • api the api object that communicates with the League of Legends server

Returns:

    a new summoner object
summoner:standardizeSummonerName (summonerName)
Given a Summoner name, put it into the League of Legends API standardized format

Parameters:

  • summonerName string the Summoner name to standardize

Returns:

    a Summoner name in the League of Legends API standardized format
summoner:getByName (name, opts)
Get a Summoner from the League of Legends API given a Summoner name

Parameters:

  • name string the Summoner name of the Summoner to retreive
  • opts a table with optional parameters:
    • expire long how long in seconds to cache a response (defaults to 24 hours, i.e. 24*60*60)
    • callback function a callback which receives the response from the API (data, code, headers) NOTE: Since you may only retreive 40 summoners at a time the callback may be called multiple times from a single call to summoner:getByName
summoner:getByNames (names, opts)
Get multiple Summoners from the League of Legends API given their Summoner names

Parameters:

  • names table an array-like table with the list of Summoner names to retreive
  • opts a table with optional parameters:
    • expire long how long in seconds to cache a response (defaults to 24hrs, i.e. 246060)
    • callback function a callback which receives the response from the API (data, code, headers) NOTE: Since you may only retreive 40 summoners at a time the callback may be called multiple times from a single call to summoner:getByNames
summoner:getById (id, opts, filter)
Get a Summoner, their runes, names, or masteries from the League of Legends API given their Summoner id

Parameters:

  • id long Summoner id of the Summoner who's data to retreive
  • opts a table with optional parameters:
    • expire long how long in seconds to cache a response (defaults to 24hrs, i.e. 246060)
    • callback function a callback which receives the response from the API (data, code, headers) NOTE: Since you may only retreive 40 summoners at a time the callback may be called multiple times from a single call to summoner:getById
  • filter string denotes what type of information to retreive, valid values: *'name' * 'masteries' * 'runes'

  • filter string denotes what type of information to retreive, valid values: *'name' * 'masteries' * 'runes'
